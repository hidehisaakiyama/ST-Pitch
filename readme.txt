ChronoPitch取扱説明書
セットアップ手順（Ubuntu / WSL 想定）

この手順は Ubuntu または WSL2 環境を想定しています。
上から順番に実行してください。

  -----------------------
  1. Git のインストール
  -----------------------

sudo apt update
sudo apt install -y git

  ----------------------------
  2. SSHキー作成（初回のみ）
  ----------------------------

ssh-keygen -t ed25519 
cat ~/.ssh/id_ed25519.pub

表示された文字列を以下に登録してください。 GitHub → Settings → SSH and
GPG keys → New SSH key

その後、以下を実行：

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519 
ssh -T git@github.com

「Hi ! You’ve successfully authenticated…」 と表示されれば成功です。

※ 引用符は半角で入力してください。

  -------------------
  3. リポジトリ取得
  -------------------

git clone git@github.com:alab-forge/ChronoPitch.git 
cd ChronoPitch

  --------------------------
  4. Python 仮想環境の準備
  --------------------------

sudo apt install -y python3-venv
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

  ----------------------------------------
  5. PostgreSQL + PostGIS のインストール
  ----------------------------------------

sudo apt install -y postgresql postgresql-contrib postgis
sudo systemctl enable --now postgresql

  ---------------------------------
  6. データベース作成（必ず実行）
  ---------------------------------

sudo -u postgres psql

psql の中で以下を実行：

CREATE USER chronopitch WITH PASSWORD 'chronopitch_pass';
CREATE DATABASE chronopitch_gis OWNER chronopitch;
\q

その後、以下を実行
CREATE EXTENSION IF NOT EXISTS postgis;

  -----------------------------------------
  7. データベース接続設定（起動前に実行）
  -----------------------------------------

export DB_NAME=chronopitch_gis
export DB_USER=chronopitch
export DB_PASSWORD=chronopitch_pass
export DB_HOST=localhost
export DB_PORT=5432

※ export コマンドは毎回起動前に実行してください。

  ---------------
  8. アプリ起動
  ---------------

source venv/bin/activate 
python run.py

ブラウザで以下にアクセス：

http://127.0.0.1:8000

  ----------
  注意事項
  ----------

・SQL文は「sudo -u postgres psql」に入った状態で実行してください。
・PostgreSQLの postgres ユーザーは使用しません。 
・DB名は chronopitch_gis を使用します。
・エラーが出た場合は、手順を上からやり直してください。




概要
ChronoPitchは、RoboCupサッカーにおけるイベントデータ（.event.csv）を対象として、
試合中に発生したプレーを属性条件および位置情報に基づいて検索し、
サッカーフィールド上で可視化・分析するWebアプリケーションである。

従来、CSVファイル単位で扱われていたイベントデータをデータベースに登録し、
Web ブラウザ上から直感的な操作で検索・可視化を行うことで、
単発プレーから連続したプレーの流れまでを一貫して分析できる基盤を提供する。

本システムは、研究用途および学習用途での利用を想定している。

---

対象データ

ChronoPitchが扱うデータは、RoboCupサッカーで公開されている
.event.csv形式のイベントデータである。

.event.csvには、パスやシュートなどのボールに関わるプレーがイベント単位で記録されており、
イベント種別、チーム情報、選手番号、発生時刻、フィールド上の座標、成功可否などが含まれる。

トラッキングデータには対応していない。

---

システム構成

ChronoPitchは、以下の構成で動作する。

バックエンド
Python
Flask
PostgreSQL
PostGIS（空間拡張）

フロントエンド
Webブラウザ
HTML / JavaScript
サッカーフィールドを模した描画画面

アップロードされたイベントデータはデータベースに保存され、
以降の検索および可視化はデータベースを基盤として実行される。

---

主な機能

イベントデータの登録
RoboCupサッカーの.event.csvファイルをアップロードし、
イベントデータをデータベースに登録する。
登録されたデータは試合単位で管理される。

試合・大会管理
登録済みの試合を大会に紐付けて管理することができる。
これにより、大会単位での検索や分析が可能となる。

単発イベント検索
イベント種別、チーム、成功可否、試合などの属性条件と、
フィールド上の指定領域（矩形選択）を組み合わせて検索を行うことができる。
検索結果は、イベントの開始地点や移動線としてフィールド上に表示される。

連続イベント検索
同一チームによるボール保持が継続しているイベント列を、
連続イベント（イベントシーケンス）として抽出することができる。
連続イベントは、他チームの割り込みやボール保持の終了を条件として区切られる。

---

インタラクティブ可視化機能

ChronoPitchの特徴として、
データベース上の検索結果とフィールド表示が相互に連動する
インタラクティブな可視化機能を備えている点が挙げられる。

検索結果として表示されたイベントやイベントシーケンスを選択すると、
対応する開始地点、終了地点、移動線、または軌跡が
フィールド上で強調表示される。

これにより、
データベース上のイベント情報と、
フィールド上の空間的な位置関係を
直感的に対応付けて確認することができる。

また、フィールド上で指定した矩形領域は、
検索条件として直接反映され、
空間条件と属性条件を統合した検索が可能である。

---

データベースについて

ChronoPitchでは、イベントデータの管理および検索のために
PostgreSQLを用いたデータベースを使用している。
位置情報を扱うため、空間拡張機能であるPostGISを導入している。

イベントデータは、
単発イベントを管理するテーブルと、
連続イベントを集約して管理するテーブルに分けて保存される。
この構成により、
個々のプレーに着目した分析から、
一連のプレーの流れを対象とした分析までを
同一のデータベース基盤上で扱うことができる。

---

基本的な利用手順

1. ChronoPitchを起動し、Webブラウザからアクセスする。
2. CSVアップロード機能を用いて.event.csvファイルを登録する。
3. 登録された試合または大会を選択する。
4. イベント検索または連続イベント検索画面で条件を指定する。
5. 検索結果をフィールド上で確認し、必要に応じて選択・比較を行う。

---

注意事項

大量の試合データを対象とするため、
検索条件によっては処理に時間がかかる場合がある。

対応データ形式は.event.csvに限定されている。
